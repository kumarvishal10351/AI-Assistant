# AGENTS.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Commands

**Setup**
```
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

**Run the app**
```
streamlit run main.py
```

> Note: `README.md` references `app.py` as the entry point, but the actual entry point is `main.py`.

**Environment**
Create a `.env` file with:
```
MISTRAL_API_KEY=your_api_key_here
```

There is no test suite, linter, or formatter configured in this project.

## Architecture

This is a single-file Streamlit application (`main.py`) — all UI, RAG logic, and LLM calls live there. There are no modules or packages to import from.

**RAG Pipeline (end-to-end)**

1. **Ingestion** (sidebar "Build Knowledge Base" button):
   - PDFs are loaded via `PyPDFLoader`, split with `RecursiveCharacterTextSplitter` (chunk size 1200, overlap 300), embedded with `MistralAIEmbeddings` (`mistral-embed`), and stored in a persistent ChromaDB instance at `chroma_db/`.
   - Re-uploading adds to the existing store rather than replacing it. Resetting deletes `chroma_db/` from disk.

2. **Query flow** (on each chat submission):
   - The user's query is first **rewritten** by the LLM for better semantic retrieval.
   - MMR retrieval (`k=6`, `fetch_k=20`, `lambda_mult=0.7`) is run against ChromaDB using the rewritten query.
   - Results are optionally filtered by document source, then re-ranked by keyword overlap and truncated to the top 4 chunks.
   - The final answer is generated by `ChatMistralAI` (`mistral-small-2506`, temperature 0.2) using a prompt that includes the retrieved context and last 6 turns of chat history.
   - A simple **confidence score** (0–100) is derived from keyword overlap between the rewritten query and retrieved chunks.

**Streamlit state & caching**
- `st.cache_resource` caches the embeddings model, LLM, and vectorstore across reruns.
- `get_vectorstore.clear()` is called after ingestion to force a reload of the updated store.
- `st.session_state` holds `chat_history` (list of `{role, content, time}` dicts) and `kb_ready` (bool).
- `st.chat_input` is used (not `st.text_input`) to avoid an infinite rerun loop — the widget auto-clears after submit.
